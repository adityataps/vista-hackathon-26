# AWS Infrastructure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Provision all AWS infrastructure for PayInvestigator — ECR, ECS Fargate, ALB, ACM, S3, IAM, CloudWatch — plus GitHub Actions CI/CD, using Terraform in `infra/`.

**Architecture:** Two Fargate services (FastAPI backend on port 8080, Nginx frontend on port 80) sit behind a single internet-facing ALB. Path-based routing sends `/api/*` to the backend and `/*` to the frontend. ACM terminates TLS at the ALB; a Cloudflare DNS-only CNAME points `vistahack26.tapshalkar.com` at the ALB. GitHub Actions uses OIDC (no long-lived keys) to build, push to ECR, and deploy via ECS rolling updates on every push to `main`.

**Tech Stack:** Terraform ≥ 1.6, AWS provider `~> 5.0`, Cloudflare provider `~> 4.0`, GitHub Actions (`aws-actions/configure-aws-credentials@v4`, `aws-actions/amazon-ecr-login@v2`, `aws-actions/amazon-ecs-render-task-definition@v1`, `aws-actions/amazon-ecs-deploy-task-definition@v2`), Docker, Python 3.12, Node 20, nginx:alpine.

## Global Constraints

- Region: `us-west-2` in all resources and CLI calls
- App name prefix: `payinvestigator` on all AWS resource names
- GitHub repo: `adityataps/vista-hackathon-26`
- Cloudflare Zone ID: `9a2b68936aec95fc2ad33a144cec981a`
- Domain: `vistahack26.tapshalkar.com`
- Backend container port: `8080`; frontend container port: `80`
- ECS services must have `lifecycle { ignore_changes = [task_definition] }` — CI owns the image tag after first deploy
- `terraform.tfvars` must never be committed — contains `cloudflare_api_token`
- All Terraform state is local (`terraform.tfstate`) — no S3 backend for this hackathon
- Cloudflare records for ACM validation: `proxied = false`, `ttl = 60`
- Cloudflare app CNAME: `proxied = false`, `ttl = 300`

```hcl
# ACM DNS validation records in Cloudflare
resource "cloudflare_record" "acm_validation" {
  for_each = {
    for dvo in aws_acm_certificate.main.domain_validation_options :
    dvo.domain_name => {
      name  = trimsuffix(dvo.resource_record_name, ".tapshalkar.com.")
      value = trimsuffix(dvo.resource_record_value, ".")
      type  = dvo.resource_record_type
    }
  }

  zone_id = var.cloudflare_zone_id
  name    = each.value.name
  content = each.value.value
  type    = each.value.type
  ttl     = 60
  proxied = false
}

# Validation waiter — Terraform blocks here until ACM issues the cert
resource "aws_acm_certificate_validation" "main" {
  certificate_arn         = aws_acm_certificate.main.arn
  validation_record_fqdns = [for r in cloudflare_record.acm_validation : r.hostname]
}
```

### Task 12: GitHub Actions CI/CD Workflow

**Files:**
- Create: `.github/workflows/deploy.yml`

**Interfaces:**
- Consumes: `github_actions_role_arn` from Terraform outputs (Task 10)
- GitHub Secrets required: `AWS_ACCOUNT_ID` — add this in the repo Settings → Secrets before pushing

- [ ] **Step 1: Add `AWS_ACCOUNT_ID` to GitHub Secrets**

```
GitHub → repo Settings → Secrets and variables → Actions → New repository secret
Name:  AWS_ACCOUNT_ID
Value: <your 12-digit AWS account ID>
```

Get the value with: `aws sts get-caller-identity --query Account --output text`

- [ ] **Step 2: Create `.github/workflows/deploy.yml`**

```yaml
name: Deploy

on:
  push:
    branches: [main]

env:
  AWS_REGION: us-west-2
  ECR_BACKEND_REPO: payinvestigator-backend
  ECR_FRONTEND_REPO: payinvestigator-frontend
  ECS_CLUSTER: payinvestigator
  BACKEND_SERVICE: payinvestigator-backend
  FRONTEND_SERVICE: payinvestigator-frontend
  BACKEND_TASK_FAMILY: payinvestigator-backend
  FRONTEND_TASK_FAMILY: payinvestigator-frontend

permissions:
  id-token: write
  contents: read

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: arn:aws:iam::${{ secrets.AWS_ACCOUNT_ID }}:role/payinvestigator-github-actions
          aws-region: ${{ env.AWS_REGION }}

      - name: Login to Amazon ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v2

      - name: Build and push backend image
        id: build-backend
        env:
          ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
          IMAGE_TAG: ${{ github.sha }}
        run: |
          docker build -t $ECR_REGISTRY/$ECR_BACKEND_REPO:$IMAGE_TAG \
                       -t $ECR_REGISTRY/$ECR_BACKEND_REPO:latest \
                       ./backend
          docker push $ECR_REGISTRY/$ECR_BACKEND_REPO:$IMAGE_TAG
          docker push $ECR_REGISTRY/$ECR_BACKEND_REPO:latest
          echo "image=$ECR_REGISTRY/$ECR_BACKEND_REPO:$IMAGE_TAG" >> $GITHUB_OUTPUT

      - name: Build and push frontend image
        id: build-frontend
        env:
          ECR_REGISTRY: ${{ steps.login-ecr.outputs.registry }}
          IMAGE_TAG: ${{ github.sha }}
        run: |
          docker build -t $ECR_REGISTRY/$ECR_FRONTEND_REPO:$IMAGE_TAG \
                       -t $ECR_REGISTRY/$ECR_FRONTEND_REPO:latest \
                       ./frontend
          docker push $ECR_REGISTRY/$ECR_FRONTEND_REPO:$IMAGE_TAG
          docker push $ECR_REGISTRY/$ECR_FRONTEND_REPO:latest
          echo "image=$ECR_REGISTRY/$ECR_FRONTEND_REPO:$IMAGE_TAG" >> $GITHUB_OUTPUT

      - name: Download backend task definition
        run: |
          aws ecs describe-task-definition \
            --task-definition $BACKEND_TASK_FAMILY \
            --query taskDefinition \
            --region $AWS_REGION > backend-task-def.json

      - name: Render backend task definition with new image
        id: render-backend
        uses: aws-actions/amazon-ecs-render-task-definition@v1
        with:
          task-definition: backend-task-def.json
          container-name: backend
          image: ${{ steps.build-backend.outputs.image }}

      - name: Deploy backend to ECS
        uses: aws-actions/amazon-ecs-deploy-task-definition@v2
        with:
          task-definition: ${{ steps.render-backend.outputs.task-definition }}
          service: ${{ env.BACKEND_SERVICE }}
          cluster: ${{ env.ECS_CLUSTER }}
          wait-for-service-stability: true

      - name: Download frontend task definition
        run: |
          aws ecs describe-task-definition \
            --task-definition $FRONTEND_TASK_FAMILY \
            --query taskDefinition \
            --region $AWS_REGION > frontend-task-def.json

      - name: Render frontend task definition with new image
        id: render-frontend
        uses: aws-actions/amazon-ecs-render-task-definition@v1
        with:
          task-definition: frontend-task-def.json
          container-name: frontend
          image: ${{ steps.build-frontend.outputs.image }}

      - name: Deploy frontend to ECS
        uses: aws-actions/amazon-ecs-deploy-task-definition@v2
        with:
          task-definition: ${{ steps.render-frontend.outputs.task-definition }}
          service: ${{ env.FRONTEND_SERVICE }}
          cluster: ${{ env.ECS_CLUSTER }}
          wait-for-service-stability: true
```

- [ ] **Step 3: Commit and push**

```bash
git add .github/workflows/deploy.yml
git commit -m "infra: GitHub Actions CI/CD — build ECR images, rolling ECS deploy"
git push origin main
```

- [ ] **Step 4: Watch the workflow**

Go to `https://github.com/adityataps/vista-hackathon-26/actions` and confirm the `Deploy` workflow runs and all steps pass (green).

Expected total runtime: ~5–8 minutes (Docker builds + ECS service stabilisation).
